# Implementation Plan: Admin Integration Settings UI

> Generated by `/PACT:plan-mode` on 2026-03-02
> Status: IMPLEMENTED

## Summary

Move integration API keys and tokens from environment variables to an admin-configurable UI. A new `integration_settings` table stores encrypted key-value pairs, with an ETS cache for fast reads and env var fallback for backwards compatibility. The admin page gets a new "Integrations" section with per-group cards, help text, and source indicators.

#### Design Approach

**Key-value table** with encrypted values. Each integration key is a row. ETS cache for read performance. PubSub for cache invalidation.

```
Admin UI (LiveView)
    │
    ▼
IntegrationSettings.put(:key, value)
    │
    ├─► Repo.insert_or_update (encrypted, in RLS-enabled transaction)
    ├─► PubSub.broadcast("integration_settings:changed", %{key: key})
    ▼
IntegrationSettings.Cache (GenServer)
    │ subscribes to PubSub
    ├─► ETS.delete(key)  [invalidate]
    ▼
Next consumer call: IntegrationSettings.get(:key)
    ├─► ETS lookup → hit → return value
    └─► ETS miss → Application.get_env(:assistant, key)  [env var fallback]
```

**Read path**: Always ETS or env var — never DB on the hot path.
**Write path**: DB write → ETS update → PubSub broadcast.
**Boot**: Cache GenServer warms ETS from DB on init (after Repo is up). Before warm completes, all reads fall through to env var — seamless.

#### Key Decisions

| Decision | Options | Recommendation | Rationale |
|----------|---------|----------------|-----------|
| Table design | KV table / Named columns / Per-integration tables | **KV table** | Extensible — new integration = new key, no migration. Project actively adds integrations. |
| Encryption | All values / Selective | **All values** | Cloak AES-GCM is fast. Avoids two-column complexity. `is_secret` controls UI masking only. |
| Cache | ETS / persistent_term / GenServer state / Application env | **ETS** | Follows Config.Loader precedent. Lock-free concurrent reads. |
| Fallback priority | DB > env var / env var > DB | **DB > env var** | Admin UI is primary config method. Env vars are escape hatch. |
| Nil semantics | Nil row = use env var / Empty string = disabled | **Both distinct** | Deleting DB row reverts to env var. Saving "" intentionally disables. |
| RLS | App-level only / Postgres RLS | **Both** | App-level is_admin as primary. Postgres RLS via session variable as defense-in-depth. |
| Goth/service account | Include / Exclude | **Exclude** | Consumed at boot before DB available. Infrastructure-level, not user-facing. |
| Help text storage | DB column / Hardcoded in Registry | **Registry** | Avoids migration for text changes. Help text is developer-authored, not admin-editable. |

## Risk Assessment

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| Boot ordering (DB not ready) | Low | Medium | Cache falls through to env var before warm completes |
| Accidental secret exposure in UI | Low | High | Backend never returns raw secrets; masked display only |
| RLS blocks legitimate admin queries | Low | Medium | Context module always sets session variable in transaction |
| Cache stale after write | Very Low | Low | PubSub invalidation is synchronous within node |
| Cloak key rotation breaks values | Low | High | Existing concern (not new) — Cloak supports multi-key rotation |
